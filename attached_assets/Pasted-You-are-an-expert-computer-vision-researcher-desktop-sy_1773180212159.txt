You are an expert computer vision researcher, desktop systems engineer, and AI software architect. Design and build a production-grade macOS desktop application called FocusLens, a privacy-first real-time telemetry assistant that improves eye contact and engagement during video calls. The application behaves like a Dynamic-Island-style floating overlay near the webcam that tracks user gaze, detects faces on the screen, relocates the active speaker near the webcam, nudges the user toward camera alignment, and records engagement telemetry with post-call analytics. The system must run fully locally with no cloud processing and must feel like a premium macOS productivity tool similar in quality to Raycast, Arc Browser, or Linear. Use Tauri with a Rust backend for the desktop runtime and Next.js + React + TypeScript + Tailwind + Framer Motion for the UI layer. The computer vision pipeline should use MediaPipe Face Mesh (468 landmarks), OpenCV, and TensorFlow Lite or ONNX Runtime, with performance-critical modules implemented in Rust and GPU acceleration via Metal on Apple Silicon. Implement a real-time gaze tracking engine that captures webcam frames, detects facial landmarks, extracts eye landmarks, estimates pupil center and eye center, calculates a normalized gaze vector, and determines camera alignment using gaze vector math where gaze_vector = normalize(pupil_center − eye_center) and gaze_angle = arccos(dot(gaze_vector, camera_vector)), with an eye-contact threshold around 10–15 degrees. Combine this with head pose estimation using solvePnP to compute yaw, pitch, and roll, and multiply the head rotation with eye vectors to produce a final gaze estimate. Maintain a pipeline capable of ~60 FPS inference with <30 ms latency. Build a screen face detection subsystem that captures screen frames using macOS ScreenCaptureKit, runs face detection using BlazeFace or RetinaFace, identifies the active speaker, crops their face, and displays it inside the overlay near the webcam so users naturally look toward the camera while watching the speaker. Implement a Dynamic Overlay Engine with transparent always-on-top windows that support compact, expanded, speaker-focus, and analytics modes, including components such as a speaker bubble, gaze indicator ring, attention progress bar, and call status indicator with smooth spring-based animations. Add a smart nudging system that subtly guides gaze when drift occurs using glow effects, directional pulses, or minimal indicators without interrupting the call experience. Create a telemetry engine storing engagement metrics locally in SQLite, capturing gaze_angle, eye_contact_ratio, gaze_drift_frequency, blink_rate, focus_duration, speaker_engagement, attention_score, and call_duration. Implement a post-call analytics dashboard that visualizes attention timelines, gaze heatmaps, engagement graphs, and improvement suggestions. Include an experimental AI eye-contact correction module capable of adjusting eye direction using a lightweight model such as a GAN-based gaze redirection network, applied only to the eye region with inference latency under ~40 ms. Add automatic meeting detection by monitoring active windows, microphone usage, and audio device activity to detect Zoom, Google Meet, Microsoft Teams, Discord, and other meeting platforms, triggering overlay activation and telemetry recording. Implement a calibration system where the user looks at multiple screen points (e.g., a 9-point grid) to build a mapping between gaze vectors and screen coordinates using regression or a lightweight neural model. Ensure multi-monitor support, correct gaze mapping across displays, and user privacy controls including pause tracking, delete telemetry, and webcam permission management. Structure the project as a modular monorepo with directories for apps (desktop UI and analytics dashboard), packages (gaze engine, face detection, overlay UI, telemetry core), native modules (Rust core and Metal kernels), and model files. Provide complete code for the vision pipeline, overlay system, telemetry infrastructure, analytics UI, and calibration workflow. Include developer tools such as debug overlays displaying landmarks and gaze vectors, performance metrics, hot reload, automated tests for gaze accuracy and latency, and benchmarking tools. Provide full build instructions, packaging scripts for macOS DMG installers, auto-update support, and deployment documentation. The final system must be optimized for low latency, minimal CPU usage (<15%), smooth GPU-accelerated inference, and a polished UI experience, resulting in a sophisticated, production-ready desktop application that improves perceived eye contact and engagement during remote meetings.
